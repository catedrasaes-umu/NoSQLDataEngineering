Abstract

Modern applications (e.g., social media, Internet of Things, and Big Data) have evidenced the limitations of relational systems to meet their scalability and performance demands. This has motivated a continuously growing number of non-relational systems with the purpose of tackling these requirements. “NoSQL” (Not SQL/Not only SQL) is used to denote this new generation of database systems (225 are listed in nosql-database.org). They are commonly classified in four major categories: document, wide column, key-value stores, and graph-based databases. A large number of companies have already embraced NoSQL databases, and adoption will rise considerably in the next years.

As the use of NoSQL grows, the need of database tools which provide functionality similar to those available for relational databases is more evident. The building of these tools must tackle the schemaless nature of most of NoSQL databases. While relational systems require the prior definition of the database schema that guides the data organization, most NoSQL databases store data without the need of having a previously defined schema. A recent report (“Insights into NoSQL Modeling”, Dataversity, 2015) has remarked that data modeling will be a crucial activity for NoSQL databases, and has also drawn attention on the need for NoSQL tools that provide three main functionalities: model visualization, code generation, and metadata management. These three kinds of capabilities are better provided with the knowledge of the database schema. Moreover, these tools should alleviate developers from the error-prone task of writing database code when the schema is not declared, but implicit into code and data. Schemaless provide a larger flexibility (e.g. non-uniform data is possible and data evolution is favored), but the advantages of using schemas are lost, e.g. schemas guarantee that data are correctly accessed. Our initial hypothesis is that NoSQL systems will continue growing in the next future, and that their construction will involve a great effort in R&D.

Our research group has recently obtained some results in the area of NoSQL Data Engineering, namely a schema inference process for document stores, and we have showed how the inferred schemas are useful to create several utilities. These results have served as a foundation for the 3-years research project here presented. The aim of the project is the design and implementation of an extensible environment and toolset to support the development and management of NoSQL-based applications. The central element of this toolset would be a unified metamodel able to represent schemas (conceptual, logical and physical) of any kind of NoSQL database, ii) metrics for optimizing the data layout and querying, and iii) batch and on-line data migration processes. The environment will support a set of basic functionality, such as: i) visualization and querying of inferred schemas, ii) code generators (e.g., code for Object-Document mappers, REST services and GraphQL), iii) assistants for database design and migration, iv) compute on-line statistics. This functionality will be modularized by the environment, and offered as building blocks of a workflow DSL that will allow users to automate high-level processes over databases.

Research challenges include the definition of a unified metamodel, visualization of large schemas, automation of the database design and migration, and the design of an extensible architecture.
